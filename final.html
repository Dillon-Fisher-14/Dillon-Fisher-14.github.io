Final Report

Problem Overview


In order to create a mesh that represents the surface of a planet (or any stretch of terrain), you need a Digital Elevation Model. A Digital Elevation Model (DEM) is a fancy word for the height map. With this height map you can write an algorithm to generate the vertices of a mesh at specific points along the surface. While DEMs for all planets and moons in our solar system are publicly available from the United State Geological Survey (USGS), not all of the DEMs have complete datasets. In fact, most of them are incomplete. USGS has only complied a complete DEM for Mars, the Moon, and Earth. All other planets and Moons have incomplete data.

The goal for this project is to take a Digital Elevation Model with incomplete data and fill it in the gaps with data that is representative of the rest of the surface.

My solution to this problem was to fit a series of sinusoidal surfaces to the image. Each surface would be a predefined pair of frequencies (one frequency for each direction), varying the amplitude and phase in each direction. The algorithm worked by fitting a sinusoidal surface to the input image, subtracting that surface from the image, and fitting a new sinusoidal surface (with a different frequency) to the new image. The algorithm does this for a specific set of frequencies (the frequencies I picked attempt to replicate a Fourier series). Once all the sine waves have been fit, I sum them all back together and the resulting surface is my output image.

Previous Work


The gap filling problem has been a long running issue in image processing. Due to sensor errors or any number of complications, raw data gather from any devise is not always be complete. Whether it is a satellite tracking weather patterns or an MRI, holes and gaps can appear in any dataset. For that reason, the gap filling problem spans many disciplines. One of the earliest solutions to this attempted to fill in gaps of sonograph data [1]. Back in 1995, they used a neural network to measure the mean and variance of the input signal of a sonograph and fill in the missing pieces. The model tracked each pixel separately, so that whenever that pixel stopped receiving data the neural network would output some value based on the stored mean and variance. This was a relatively simple example to a paper called High-Resolution Image Inpainting using Multi-Scale Neural Patch Synthesis [2]. This paper described a deep neural network model that takes an input image (with no missing data) and any arbitrary mask, outputting an image with the contents under the mask removed and filled in by the neural network. The model worked with varying degrees of success.

While it is uncommon now, many papers on gap filling in the past used traditional image processing methods instead of machine learning. One paper published result of their program filling in gaps of weather data using Singular Spectrum Analysis [3]. There are a ton of papers that use Fourier Analysis and similarly well-developed methods to analyze image and fill in the gaps.

[1] H. Klebaek, J. A. Jensen and L. K. Hansen, "Neural network for sonogram gap filling," 1995 IEEE Ultrasonics Symposium. Proceedings. An International Symposium, 1995, pp. 1553-1556 vol.2, doi: 10.1109/ULTSYM.1995.495851.
[2] Chao Yang, Xin Lu, Zhe Lin, Eli Shechtman, Oliver Wang, Hao Li, “High-Resolution Image Inpainting Using Multi-Scale Neural Patch Synthesis,” Proceedings of the IEEE Conference on Computer Vision and Pattern Recognition (CVPR), 2017, pp. 6721-6729
[3] Ghafarian Malamiri, Hamid Reza, et al. "Gap-filling of MODIS time series land surface temperature (LST) products using singular spectrum analysis (SSA)." Atmosphere 9.9 (2018): 334.

By the second checkpoint I created dropout tests for my 1-dimensional algorithm and updated it to work in 2-dimensions. Or at least I thought I had a working 2-dimensional version working.

I ran the algorithm on a larger image and the results were not as good as I had hoped. A lot of artifacts here and there that we implicative of bugs in the program. I don’t remember all the changes between checkpoint 2 and the final demo, but it was a lot. I probably spent twice as much time working on this project after checkpoint two as compared to before. A lot of the progress I made after check point two was optimizing the program, fixing all of the bugs in my initial version, adding heuristics and running tests. I will speak a little bit towards each of those.

I used the programming language python for this project. The reason for this was to easily produce diagrams and videos for the report. About halfway through I had wished that I had written it in a more optimized language (like C++) and almost rewrote the whole project. I decided on some middle ground and rewrote the project to cython (compiled python). Two of my lab-mates helped me convert the code to cython and add a few python specific optimizations that I am very grateful for. Just to be clear. They are however, very interested in this project as we will be continuing it together next semester. But regardless, I was able update the code with a huge 5-8 times performance increase.

Once the program was optimized, I started running tests on larger images and this is when I realized that the algorithm did not work as well as I thought it did. The Fourier series approximation that I was generating was not approximating the input data. For example, if I gave it something with no gaps in it, what it returned was not very close to what I input. The general shape was the same, but a lot of the details were lost. Also, the program only worked on a square image. These were all relatively small bumps in the road of this project, but there were quite a few small bugs here and there to handle.

After being fairly certain that the program was at least working I started looking towards heuristics to make it better. There were a bunch of small heuristics I added which altered how well the algorithm performed. Things like how I initialized the amplitude and phase of the first sine wave in gradient descent. The most important heuristic I added was changing the order of the frequencies of each sine wave that I fit. This was a lot more impactful for the two-dimensional case where I have two frequencies to keep track of. I decided against having two nested for loops that loop through each direction’s frequency respectively. To make this clear, if I have three frequencies that I was to run in each direction I would loop through the frequencies in this order [(1,1), (1,2), (1, 3), (2, 1), (2, 2), (2, 3), (3, 1), (3, 2), (3,3)]. I found that the Fourier approximation fits better if I do all of the lower frequencies first. I don’t know how to fully articulate what this ordering would be for these frequency pairs, but here is what the order looks like for our three frequency example: [(1, 1), (1, 2), (2, 1), (1,3), (2, 2), (3, 1), (2,3), (3, 2), (3, 3)]

The remainder of the development of this project was bring back all my changes to the 1-dimensional case, running tests, and generating pretty images and graphics for this report/presentation.

Did I accomplish anything that has never been done before?


My project does not do anything new. People have been using signal analysis in image processing for decades, and recently machine learning has had great results in the gap fill problem. There are a ton of papers on similar models that perform extremely well. Having said that, I would be surprised if there is a paper out there that does exactly what my algorithm does (I could not find it if it exists).

While writing this report I found a paper called “Gap-filling for the high-resolution PET sinograms with a dedicated DCT-domain filter” which was theoretically close to what I did.

How well did I meet the goals of my project?


I am very happy with the results of my project. I have had the broad idea to use Fourier analysis in order to fill in terrain data for a few years now. It was really fulfilling to give it a go and see what happens.

I met the main goal of my project, which was to take an input DEM with gaps and to output a DEM without gaps. However, I did not have time to use the produced DEM and see what it looks like projected to a sphere. While it was my plan to originally do that, when it came down to a time crunch at the end of the semester, I opted to run two-dimensional dropout tests instead since I felt they would give a better representation of my work.

What are the next steps for this project?


This is something I am super excited about. I have talked with a few of my friends (lab-mates) and we plan to work together and take this project to the next level next semester.

The base algorithm can be applied to data in n-dimensional space. I really want to run some simple tests on 3-dimensional volumes to see if I can get similarly gap filling results. Since this algorithm doesn’t have the primary limitations of a typical Fourier series (the data points don’t all have to be equally spaced) I am excited to see how well it works in the future.

Along with that, I have been looking into Entropy Maps. They are basically a filter on an image that shows areas of volatility in an image. For a planet that corresponds separating out areas that are relatively flat/smooth from areas with a lot of geologic activity. In using this I think I can tune how much I fit specific data. For example, the smoother data will have a tighter fit (smaller fit tolerance) than the more chaotic data.
